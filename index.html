Orlando Source Software

tl;dr I want us to make a venue for long-term project-based collaboration between developers of different skill levels.

For Juniors

It's Hard to Learn Without Guidance

You graduated from a four-year program, a two-year program, or a bootcamp, or maybe you have been teaching yourself.

You want to keep learning, but the options seem limited:

  • Tutorials all feel the same - you learn new syntax and hot frameworks, but not how to build software at scale
  • Q+A sites lack context - it can be hard to get a straight answer when other developers don't know what you are trying to do
  • Building projects is great - but how do you know you're doing it right?

Employers Want Experience

Teams want to hire developers with the proven ability to

  • Contribute to a large-scale project
  • Function as part of a team
  • Work with limited supervision

And there's just not enough time in a bootcamp or degree program to demonstrate those abilities to the degree many employers want.

For Seniors

It's Hard to Help Without Context

You have been working as a developer for a while, and you have learned a lot.

You want to help new developers, but how?

  • Debugging is not enough - Stack Overflow and Slack are great for quick answers, not for the big picture
  • Software is built over time and yours is too limited to learn about and supervise every project out there - especially when you have projects of your own that need work

It's Hard to Hire Without Context

Everybody knows hiring is a mess - a long, slow process often using metrics that may not even weakly correlate with on-the-job performance. Ideally, you would hire only people who you know and trust to be able to contribute to a large-scale project as part of a team with limited supervision - but all of those people have jobs. The community needs a way to evaluate junior talent so we can help junior developers become working developers as fast as possible.

Goals of this Organization

This organization is to be a place for developers to help each other get better. Projects contributed by members of the Orlando Developer community will be built out over time to give juniors the experience of contributing to large-scale codebases. We will leverage the existing strengths of the Orlando Developer community and of GitHub:

  • ODev feedback is quick and friendly - if you have questions about a project, you'll be able to find maintainers in Slack and/or the local Meetups. That way you can learn what you need with less delay and stress than posting in a public forum like GitHub.
  • GitHub provides context and history - conversations in person or on Slack are great unless you need records of them later. GitHub preserves both the conversations people have about each project in the issues, and the code itself, which makes long-term communication about a project possible.

Interested? We Need You!

This organization has everything it needs to succeed except for two things: people and projects.

If you would like to join in, either by offering a personal project for others to work on, volunteering to be a project maintainer, and/or working on projects, please open an issue at the expressing interest repo.

Properties of Good Projects

While I'm sure this list (and the whole organization) will evolve, these are my current thoughts about what kind of project would be a good fit for us:

  • Serve a Clear Purpose - both inexperienced developers who may contribute code, and seniors who will review junior work, will need be able to figure out what the project does.
  • Have Some Code - to give developers the experience of dealing with an unfamiliar, pre-existing codebase.
  • Have (GitHub) Issues - to guide initial contributions, suggest ways you would like to see the project extended.

I added one of my own projects, a React application I wrote as a way to teach myself React. It's not my best code, but what legacy code is? The point is provides a basis for other developers to contribute to an unfamiliar code base. If you are a junior and you would like to take a stab at a GitHub issue for the first time, awesome! Go fork the project and let me know if an issue is unclear. If you are one of our local JS/React stars and you'd like to suggest changes in the project architecture, sweet! I want to learn too.
